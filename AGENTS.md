# AGENTS.md

## Project Overview

- Bun workspace for publishable OpenCode plugins in `packages/*`.
- Current packages:
  - `packages/opencode-codebase-memory/` - wires `codebase-memory-mcp` into OpenCode and can auto-index the active project.
  - `packages/opencode-improve/` - injects the `/improve` command and bundled audit-planning references.
  - `packages/opencode-omniroute-auth/` - OmniRoute auth/provider plugin. Read its nested `packages/opencode-omniroute-auth/AGENTS.md` before making package-local changes.
  - `packages/opencode-zellij/` - Zellij pane-management plugin.
- Root `README.md` is generated by `bun run readme` from package manifests and `scripts/update-readme.ts`. Do not hand-edit it.

## Source Of Truth

- Workspace scripts and Bun version: `package.json`
- Canonical lint/check tasks: `mise.toml`
- CI order and required checks: `.github/workflows/ci.yaml`
- Release automation: `.github/workflows/release.yaml`, `release-please-config.json`, `.release-please-manifest.json`
- Plugin smoke helper: `scripts/opencode-smoke.mjs`
- Package-level build/test entrypoints: `packages/*/package.json`

## Setup

- Use Bun `1.2.23` (`packageManager` in `package.json`, mirrored in `mise.toml` and CI).
- Use Bun for workspace development. `bunx only-allow bun` blocks `npm install` at the repo root.
- Recommended bootstrap:

```bash
mise install
bun install --frozen-lockfile
```

## Commands

| Purpose | Command | When to run |
| --- | --- | --- |
| Install deps | `bun install --frozen-lockfile` | Fresh checkout or lockfile changes |
| Lint | `mise run lint` | After TS/JS edits |
| Build all packages | `bun run build` | After source edits; this is also the repo's closest thing to a root typecheck |
| Test all packages + root smoke tests | `bun run test` | Before handoff for code changes |
| Full local validation | `mise run check` | Preferred final check; runs lint first, then `bun run check` |
| Regenerate generated README | `bun run readme` | After package add/rename/description changes |
| Test one package | `bun run --filter @auron-labs/opencode-improve test` | Focused iteration inside one package |
| Run only the root smoke unit tests | `node --test scripts/opencode-smoke.test.mjs` | When changing `scripts/opencode-smoke*.mjs` |
| Preview plugin smoke usage | `bun run smoke:plugin -- --help` | To inspect isolated smoke-test inputs |

## Testing Notes

- Each package `test` script builds first, then runs `node --test test/*.test.mjs`.
- Package tests import `../dist/index.js`; if you bypass the package script and run `node --test` directly, build the package first.
- Root `bun run test` expands to `bun run --filter '*' test && node --test scripts/opencode-smoke.test.mjs`.
- `scripts/opencode-smoke.mjs` prints an isolated `opencode run` command; it does not execute the smoke run itself.
- The smoke helper requires a built `dist/index.js` for the selected plugin, and actual smoke execution requires `opencode` on `PATH`.

## Change-Aware Validation

- Default final check: `mise run check`.
- If you only touch one package, start with `bun run --filter <package-name> test`.
- If you change package metadata that feeds the package table or descriptions, run `bun run readme` and review `README.md`.
- If you change smoke tooling, run `node --test scripts/opencode-smoke.test.mjs` even if package tests already pass.
- If you change the upstream sync script, run `node --test scripts/sync-improve-upstream.test.mjs`. The script discovers files via the GitHub tree API and auto-deletes stale locals in `packages/opencode-improve/references/`; preserve both behaviors.
- If you change `packages/opencode-omniroute-auth/`, also follow its nested `AGENTS.md` for package-specific validation and release constraints.

## Generated And Ignored Files

- `README.md` is generated by `scripts/update-readme.ts`.
- `packages/*/dist/` is build output and ignored by git; do not edit generated files in `dist/`.

## Repo Conventions

- Packages are independently versioned and released from `packages/*` via release-please.
- Keep edits scoped to the relevant package; shared root logic lives mostly in `scripts/`, workspace manifests, and release config.
- No repo-wide formatter config is checked in. Match the surrounding file style and avoid unrelated formatting churn.

## Release Notes

- CI runs `bun install --frozen-lockfile`, `bun run lint`, `bun run build`, and `bun run test`.
- Release publishing is automated from `main`/`master` by `release-please`; the publish job builds and publishes only the package paths selected by the release job.
- Local release PR preview command from `CONTRIBUTING.md`:

```bash
bunx release-please manifest-pr --config-file release-please-config.json --manifest-file .release-please-manifest.json --target-branch main
```

## Security

- Do not commit credentials or private config.
- Report vulnerabilities per `SECURITY.md` instead of opening a public issue.
